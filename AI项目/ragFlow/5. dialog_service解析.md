# dialog_service.py 讲解

## 一、文件职责

`dialog_service.py` 是 **对话/问答业务核心**，负责：

- 纯 LLM 对话（无知识库）
- RAG 对话（有知识库：检索 → 拼 prompt → 调 LLM）
- 插件/CRM 对话（文案优化、翻译、辅助填报等）

## 三、主流程：`chat()` 函数（约 378–632 行）

`chat(dialog, messages, stream=True, **kwargs)` 是 RAG 对话主入口，整体流程如下。

### 3.1 前置与模型准备

```python
# 1. 取最近 3 轮用户问题
questions = [m["content"] for m in messages if m["role"] == "user"][-3:]

# 2. 多轮问题改写（若开启 refine_multiturn）
if len(questions) > 1 and prompt_config.get("refine_multiturn"):
    refine_result = refine_and_categorize_question(...)  # 调用 LLM 改写 + 分类
    questions = [refined_question]
    category = output_json.get("category")  # 用于 CRM 分类、模板回复等

# 3. 问题分类（categorize_question）
category = categorize_question(dialog, questions[-1], ...)

# 4. 人工模板回复：某些 category 直接返回配置好的话术
if category in manual_reply_template:
    yield {"answer": 模板内容, ...}
    return

# 5. 按 category 刷新对话配置（refresh_dialog_byCategory）
dialog = refresh_dialog_byCategory(...)
```

### 3.2 无知识库分支：`chat_solo`

```python
if not dialog.kb_ids and not dialog.prompt_config.get("tavily_api_key"):
    for ans in chat_solo(dialog, messages, stream, **kwargs):
        yield ans
    return
```

无知识库且无 Tavily 时，直接走 `chat_solo` 做纯 LLM 对话。

### 3.3 有知识库分支：RAG 流程

```
1. get_models(dialog)        → embd_mdl, chat_mdl, rerank_mdl, tts_mdl
2. 跨语言 / 关键词扩展       → cross_languages(), keyword_extraction()
3. retriever.retrieval()     → 召回相关 chunks
4. 可选：Tavily、知识图谱    → kbinfos["chunks"].extend(...)
5. knowledges = kb_prompt()  → 拼成知识库文本
6. kwargs["knowledge"] = ... → 填入 system prompt
7. message_fit_in()          → 控制 history 长度
8. chat_mdl.chat_streamly()  → 流式生成
9. decorate_answer()         → 引用整理、耗时统计、Langfuse 等
```

### 3.4 关键检索调用

```python
kbinfos = retriever.retrieval(
    " ".join(questions),
    embd_mdl,
    tenant_ids,
    dialog.kb_ids,
    page=1,
    page_size=dialog.top_n,
    similarity_threshold=dialog.similarity_threshold,
    vector_similarity_weight=dialog.vector_similarity_weight,
    doc_ids=attachments,        # 可限定文档
    top=dialog.top_k,
    rerank_mdl=rerank_mdl,
    rank_feature=label_question(...),
)
```

这里就是与 `rag/nlp/search.py` 中的 `Dealer.retrieval` 对接的地方。

## 四、核心函数说明

### 4.1 `chat_solo()`（约 141–174 行）

- **用途**：纯 LLM 对话，不走知识库检索。
- **参数**：`dialog`、`messages`、`stream`、`kwargs`（含 `language`、`category` 等）。
- **流程**：用 `prompt_config["system"].format(**kwargs)` 构建 system prompt，再调 `chat_mdl.chat_streamly()` 或 `chat()`。
- **输出**：通过 `yield` 返回 `{"answer", "reference", "audio_binary", "prompt", ...}`。

### 4.2 `plugin_chat_solo()`（约 176–408 行）

- **用途**：插件场景的纯 LLM，不走知识库。
- **source_type 含义**：
  - 5：文案优化
  - 6：翻译
  - 7：对话总结
  - 8：邮件总结
  - 9：AI 识图
  - 10：辅助填报（含 CRM 查询、产品信息等）
- **流程**：根据 `source_type` 选择不同 prompt 模板，再调用 LLM。

### 4.3 `get_models()`（约 318–337 行）

```python
def get_models(dialog):
    # 从绑定的知识库获取 embedding 模型
    embd_mdl = LLMBundle(dialog.tenant_id, LLMType.EMBEDDING, embedding_list[0])
    # Chat 模型
    chat_mdl = LLMBundle(dialog.tenant_id, LLMType.CHAT, dialog.llm_id)
    # Rerank 模型（可选）
    rerank_mdl = LLMBundle(...) if dialog.rerank_id else None
    # TTS 模型（可选）
    tts_mdl = ...
    return kbs, embd_mdl, rerank_mdl, chat_mdl, tts_mdl
```

负责按 `dialog` 配置组装各模型实例，供后续检索和生成使用。

### 4.4 `decorate_answer()`（约 537–609 行）

- 拆分 `
` 内容（思考过程 vs 正文）。
- 解析 `[ID:n]` 引用，整理 `reference`。
- 调用 `repair_bad_citation_formats()` 统一引用格式。
- 统计耗时、写入 Langfuse。
- 返回 `{"answer", "reference", "prompt", "category", "total_time_cost", ...}`。

### 4.5 `use_sql()`（约 834–917 行）

- **用途**：Text2SQL，用 LLM 根据问题生成 SQL，在 ES/Infinity 中执行。
- **流程**：LLM 生成 SQL → `retrievaler.sql_retrieval()` → 转为 Markdown 表格返回。
- **当前状态**：在 `chat()` 中对应逻辑被注释，不参与主流程。

## 六、关键配置来源

| 配置项 | 来源 | 用途 |
|--------|------|------|
| `dialog.kb_ids` | 对话配置 | 决定是否走 RAG |
| `dialog.llm_id` | 对话配置 | 选择 Chat 模型 |
| `dialog.top_n` | 对话配置 | 召回 chunk 数量 |
| `dialog.prompt_config` | 对话配置 | system prompt、多轮改写、跨语言等 |
| `kwargs` | 调用方 | `language`、`region`、`source_type`、`doc_ids` 等 |

## 七、与 Java 的对应关系

| Python | 对应 Java 概念 |
|--------|----------------|
| `yield {...}` | `Stream<T>` 或 `Iterator` 流式返回 |
| `**kwargs` | `Map<String, Object>` 或 可选参数 |
| `chat_mdl.chat_streamly()` | 异步流式调用外部 LLM API |
| `retriever.retrieval()` | 调用检索服务，类似 `SearchService.search()` |
